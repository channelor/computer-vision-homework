# “看图说话切东西”助手系统设计方案

## 1. 系统目标

本方案设计一个多模态图像交互助手，用户可以上传图片并输入自然语言指令，例如“把图中的苹果切出来”“找出所有车并说明结果”“这个区域是什么物体”。系统需要返回分割结果、类别判断、数量估计和自然语言解释。

该系统的核心思想是结合 SAM、CLIP 和 LLM：SAM 负责分割，CLIP 负责开放词表语义匹配，LLM 负责理解用户指令、组织推理流程并生成解释。

## 2. 任务能力

系统需要具备以下能力：

- 图像分割：根据点提示、框提示或自动候选区域生成 mask。
- 语义识别：判断候选区域是否符合用户描述的目标类别。
- 语言理解：解析用户的自然语言指令，决定需要分割、识别、计数还是解释。
- 交互修正：当结果不确定时，向用户追问或允许用户补充点、框提示。
- 结果解释：用自然语言说明识别依据、分割结果和不确定性。

## 3. 系统流程

完整流程如下：

```text
用户上传图片和指令
        ↓
LLM 解析指令，提取目标类别和任务类型
        ↓
生成提示方式：点提示、框提示、文本定位或自动候选区域
        ↓
SAM 生成候选分割 mask
        ↓
CLIP 对候选区域和文本类别进行相似度匹配
        ↓
后处理模块过滤重复 mask、低置信度 mask 和异常区域
        ↓
LLM 汇总视觉模块结果，生成自然语言回答
        ↓
用户可继续点击、画框或修改指令进行交互修正
```

例如用户输入“把图中的苹果分割出来”，LLM 先识别目标类别为“苹果”，系统可以先通过文本定位模块或人工框提示获得候选区域，再调用 SAM 生成 mask，最后用 CLIP 判断候选区域是否与“apple”语义匹配。

## 4. 模块设计

### 4.1 LLM 指令解析模块

输入：用户自然语言指令。

输出：结构化任务描述，例如：

```json
{
  "target": "apple",
  "task": "segment",
  "need_count": true,
  "interaction": "box_or_point"
}
```

该模块负责处理用户表达中的歧义，例如“切东西”可能表示分割目标，“这个是什么”可能表示识别区域，“有几个苹果”则表示计数任务。

### 4.2 SAM 分割模块

输入：图像、点提示、框提示或自动候选提示。

输出：mask、score、面积、边界框。

SAM 适合生成高质量 mask，但它不直接理解类别，也不保证一个 mask 对应一个实例。因此在系统中，SAM 更适合作为候选区域生成和边界细化模块。

### 4.3 CLIP 语义匹配模块

输入：候选区域图像、文本类别 prompt。

输出：图文相似度分数和类别排序。

CLIP 可以判断候选区域是否接近用户目标类别，例如判断某个 mask 区域是否更像“apple”“orange”还是“ball”。为了提高稳定性，可以使用 prompt ensemble，例如：

- `a photo of an apple`
- `a cropped photo of an apple`
- `a close-up photo of an apple`

### 4.4 后处理模块

后处理负责把 SAM 和 CLIP 的输出变成稳定结果，包括：

- 去除面积过小或过大的异常 mask。
- 合并高度重叠的候选区域。
- 使用 CLIP 分数过滤语义不匹配区域。
- 对计数任务进行非极大值抑制或连通域分析。
- 保留不确定结果，交给用户确认。

### 4.5 结果生成模块

LLM 根据视觉模块输出生成最终回答，例如：

> 已分割出图中的苹果。当前结果包含 1 个主要目标，mask 边界覆盖苹果主体和部分果柄。由于阴影区域与苹果边缘接近，右下角边缘可能存在轻微误分割。

## 5. 难点与解决方案

| 难点 | 原因 | 解决方案 |
| --- | --- | --- |
| 密集小目标计数不稳定 | SAM 不负责实例检测，目标可能遮挡或粘连 | 结合检测模型、NMS、面积过滤和人工校正 |
| CLIP 类别混淆 | 相似物体文本和视觉特征接近 | 使用 prompt ensemble，限制候选类别，引入阈值 |
| SAM mask 溢出 | 点提示约束弱，边界受阴影或背景影响 | 使用框提示、负点提示或多轮交互修正 |
| 用户指令不明确 | 自然语言可能省略目标或操作 | LLM 主动追问，例如“你想分割哪个物体？” |
| 多模型误差累积 | SAM、CLIP、LLM 各自可能出错 | 输出置信度和中间结果，让用户可检查和修正 |

## 6. 结合本次实验的设计启发

本次 CLIP 零样本分类在 CIFAR-10 上达到 93.38% 准确率，说明 CLIP 可以作为开放类别识别模块。但 CLIP 在 cat/dog、deer/horse 等相似类别上仍有混淆，因此系统不能只依赖 CLIP 做最终判断。

本次 SAM 实验表明，点提示和框提示都可以分割单个苹果，但框提示更稳定。对于“切东西”这类任务，如果用户能提供框提示，系统应优先使用框提示；如果用户只点击一点，系统应展示多个候选 mask 供用户确认。

因此，一个实用系统应采用“LLM 组织任务 + SAM 生成 mask + CLIP 语义验证 + 用户交互修正”的组合方式，而不是让单个模型独立完成所有功能。

## 7. 可扩展方向

- 加入 Grounding DINO，实现文本到目标框的自动定位。
- 使用 SAM2 支持视频或连续帧分割。
- 加入实例分割和目标检测模型，提高计数任务可靠性。
- 建立交互式前端，让用户直接点击、画框、删除错误 mask。
- 对低置信度结果进行主动追问，提高系统可控性。

## 8. 总结

“看图说话切东西”助手需要多个模型协作完成。SAM 擅长分割但不理解类别，CLIP 擅长开放语义匹配但不输出精确边界，LLM 擅长理解指令和生成解释但不能直接处理像素级分割。三者结合可以形成较完整的图像交互系统。实际应用中，还需要后处理、置信度控制和用户交互机制来减少错误。
