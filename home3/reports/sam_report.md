# SAM 交互式分割分析报告

## 1. 任务目标

本实验体验 Segment Anything Model（SAM）的交互式分割能力，重点比较点提示和框提示两种交互方式在同一张图像上的分割效果，并讨论 SAM 是否适合直接用于密集苹果计数任务。

## 2. 实验设置

| 项目 | 内容 |
| --- | --- |
| 输入图像 | `inputs/apple.jpg` |
| 图像内容 | 白色背景下的单个苹果 |
| SAM 模型 | `sam_vit_b` |
| Checkpoint | `sam_vit_b_01ec64.pth` |
| 点提示 | `[1550, 1250]`，位于苹果主体中心附近 |
| 框提示 | `[520, 260, 2500, 2210]`，覆盖苹果主体 |
| 点提示输出 | `outputs/sam_demo/point_prompt/` |
| 框提示输出 | `outputs/sam_demo/box_prompt/` |

实验均开启 `multimask`，即 SAM 对同一提示返回多个候选 mask，并给出对应的预测分数。

## 3. 点提示结果分析

点提示使用苹果主体中心附近的一个正点：`[1550, 1250]`。SAM 返回三个候选 mask，主要结果如下：

| Mask | Score | 面积占比 |
| --- | ---: | ---: |
| mask_0 | 1.0007 | 41.67% |
| mask_1 | 1.0032 | 40.68% |
| mask_2 | 0.6705 | 21.24% |

从可视化结果看，点提示可以成功定位苹果主体，并分割出苹果的大部分区域。`mask_0` 和 `mask_1` 面积接近，均覆盖苹果主体和部分果柄；`mask_2` 面积明显偏小，说明点提示存在一定歧义，SAM 会给出不同尺度或不同边界解释。

点提示的优点是交互成本低，只需要用户点击目标内部一点即可得到候选分割结果。缺点是提示信息较少，如果目标边界复杂、存在阴影或背景干扰，模型可能把部分背景、阴影或附属结构一起分割进去。本实验中点提示结果在苹果边缘附近有轻微溢出，但整体分割质量较好。

## 4. 框提示结果分析

框提示使用 `[520, 260, 2500, 2210]`，基本覆盖苹果主体。SAM 返回三个候选 mask：

| Mask | Score | 面积占比 |
| --- | ---: | ---: |
| mask_0 | 0.9962 | 40.75% |
| mask_1 | 0.9978 | 40.61% |
| mask_2 | 0.8850 | 40.51% |

框提示结果比点提示更稳定。三个候选 mask 的面积占比非常接近，都在 40.5% 到 40.8% 附近，说明框提示给出了更明确的目标范围，减少了 SAM 对目标尺度的歧义。可视化结果显示，框提示基本锁定苹果主体，背景误分割较少，边界也更稳定。

框提示的优点是目标范围明确，适合复杂背景、多目标或目标边界不清楚的场景。缺点是人工输入成本高于点提示，需要用户画出一个合理边界框。如果框过大，可能引入背景；如果框过小，可能漏掉目标边缘。

## 5. 点提示与框提示对比

| 维度 | 点提示 | 框提示 |
| --- | --- | --- |
| 操作成本 | 低，只需点击一点 | 较高，需要画框 |
| 目标约束 | 较弱 | 较强 |
| 结果稳定性 | 中等，候选 mask 差异较大 | 更稳定，候选 mask 面积接近 |
| 适合场景 | 单个明显目标、快速交互 | 复杂背景、边界需要约束的目标 |
| 本实验表现 | 成功分割苹果，但有轻微歧义 | 成功分割苹果，边界更稳定 |

综合来看，点提示适合快速粗分割，框提示适合需要更稳定结果的分割任务。本实验中两者都能分割苹果，但框提示的候选结果更一致，因此在报告分析中可以认为框提示更稳定。

## 6. SAM 是否适合数苹果

SAM 不适合直接作为“数苹果”的完整解决方案。SAM 的核心能力是根据提示生成分割 mask，而不是自动检测并统计实例数量。对于单个苹果，SAM 可以很好地完成分割；但在密集苹果场景中，会出现以下问题：

- 多个苹果相互遮挡，边界粘连，SAM 可能把多个苹果分成一个整体。
- 小目标或远处目标提示不足，容易漏分割。
- 光照、阴影和颜色相似区域可能导致 mask 溢出。
- SAM 本身不输出稳定的实例计数逻辑，需要额外后处理。

更合理的方案是：先用检测模型或 Grounding DINO 根据文本找到候选苹果框，再用 SAM 对每个候选框细化 mask，最后结合非极大值抑制、面积过滤、连通域分析或人工校正进行计数。也可以将 SAM 作为交互式工具，由用户逐步修正漏检或误检目标。

## 7. 总结

SAM 在本实验中成功完成了苹果图像的交互式分割。点提示操作简单，能够快速定位目标；框提示约束更强，分割结果更稳定。对于单目标分割任务，SAM 表现较好；但对于密集苹果计数，SAM 不能单独完成完整任务，需要结合检测、实例分割后处理或人工交互机制。
